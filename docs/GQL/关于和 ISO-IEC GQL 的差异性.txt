GQL相关文档
相关语法请参考下面三个 repo：
https://github.com/opengql/grammar
https://github.com/opengql/railroad
https://github.com/opengql/editor
https://opengql.github.io/railroad/

1. 不支持 Graph Type；直接创建 Graph 的时候同步生成匿名 Graph Type，简化使用；
语法参考：
CREATE GRAPH IF NOT EXISTS TRON {
NODE A {Address String PRIMARY KEY, Label SET<enum>},
EDGE B (A)-[{tx string PRIMARY KEY, sum float,times int}]->(B)
}
2.修改 Graph的数据类型方式为：
3.ALTER GRAPH TRON {
            ADD NODE C  {Address String PRIMARY KEY, Label SET<enum>},
            ALTER EDGE B ADD PROPERTIES RISK_LABEL ENUM,
}
3.扩展GQL支持主键，通过主键进行查询能获得更高的效率；主键在同一个点类型内唯一；边也有主键，边的主键在边类型内唯一；点上主键是必须的。边上主键不是必须的，但是如果有主键的话需要确保唯一性；
4. 支持内置点类型和边类型；内置的点和边的类型，前缀用__表示，非内置的点和边类型，不允许使用__
5. 不支持 Node/Edge Label/key label set 等；简化建模复杂度；
6. 支持 SET 数据类型和枚举类型，用 SET<枚举>来存放web3 的各种标签，同时提供各种函数，用来高效的增删改相关内容；
7. 不支持无向边，暂无场景；
8. 仅支持 closed graph，不支持 open graph
